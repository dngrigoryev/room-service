
# Сервис бронирования переговорок

  

REST API сервис для управления переговорками. Разработан на Go с применением принципов слоистой архитектуры и использованием PostgreSQL.

  
## Используемый стек

  

-  **Язык:** Go 1.25

-  **База данных:** PostgreSQL 17

-  **Аутентификация:** JWT (JSON Web Tokens)

-  **Инфраструктура:** Docker, Docker Compose



## Ключевые аспекты

  

-  **Ролевая модель (Пользователь / Администратор):** Разделение прав доступа. Добавление новых комнат и настройка расписания доступны только администраторам.

-  **Ленивая генерация слотов:** Временные слоты (по 30 минут) генерируются динамически на основе заданного расписания комнаты только в тот момент, когда пользователи запрашивают свободные места на конкретную дату. За счёт этого удаётся существенно экономить место в БД.

-  **Аутентификация:** Взаимодействие с API посредством JWT-токенов.

  

## Запуск проекта


  

1.  **Клонируйте репозиторий:**

```bash

git clone github.com/dngrigoryev/room-service

```

  

2.  **Запустите проект и все зависимости (базу данных):**

```bash

make up

```

*Сервис будет доступен по адрeсу: http://localhost:8080*

  

3.  **(Опционально) Заполните базу данных тестовыми данными:**

```bash

make seed

```

*Создаст пользователей, комнаты и расписания для тестирования.*

  

## Эндпоинты

  

-  **Авторизация (`/auth`)**

- Регистрация и логин пользователей. Получение токена.

-  **Комнаты (`/rooms`)**

- Получение списка всех переговорных комнат.

- Создание новой комнаты (только Admin).

-  **Расписание (`/schedules`)**

- Настройка графика работы конкретной переговорки (только Admin, один раз для комнаты).

-  **Слоты (`/slots`)**

- Получение доступных для бронирования слотов на указанную дату (ленивая генерация).

-  **Бронирование (`/bookings`)**

- Создание брони на выбранный слот.

- Получение списка своих бронирований.

- Отмена бронирования.
